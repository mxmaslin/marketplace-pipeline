# Тестовое задание: Пайплайн аналитики маркетплейсов с ИИ-классификацией и интеграцией с CRM

##  Бизнес-контекст
Компании необходимо автоматизировать процесс мониторинга конкурентов и поиска нишевых товаров. Мы собираем каталоги маркетплейсов, с помощью ИИ определяем ценовой сегмент товаров (чтобы не анализировать «премиум» и «эконом» в одной куче) и автоматически ставим задачи менеджерам в CRM для проработки самых интересных позиций.

## Выбор стека (на усмотрение кандидата)
*   Маркетплейс (источник данных, выбрать 1): Ozon, Wildberries, Яндекс.Маркет, Авито.
*   CRM (приемник задач, выбрать 1): AmoCRM или Битрикс24.
*   LLM (для классификации, выбрать 1): OpenAI, Anthropic, YandexGPT, GigaChat или ....

##  Функциональные требования

### 1. Парсинг маркетплейса
*   Выбрать одну произвольную категорию товаров.
*   Собрать 10 000 товаров. 
*   Реализовать сбор, пагинацию, обработку rate-limit (429 Too Many Requests), экспоненциальные повторные попытки (retries) и graceful degradation (если маркетплейс начал отдавать ошибки, пайплайн не должен падать, а должен сохранять то, что успел собрать).
*   *Опционально:* предусмотреть DEMO_MODE=true, чтобы для локальных тестов собирать не 10к, а 50-100 товаров.

### 2. Нормализация и обогащение данных (LLM)
*   Привести данные к единой схеме (Pydantic): id, name, price, currency, url, category, collected_at, description, ...
*   Этап ИИ-классификации: Прогнать названия и описания товаров через LLM для присвоения ценового/потребительского сегмента. 
    *   Сегменты (строго 3 варианта): Эконом, Стандарт, Премиум.
    *   *Требование к архитектуре:* 10 000 отдельных запросов к LLM — это дорого и долго. Кандидат должен реализовать батчинг (отправку пачек товаров в одном промпте) или использовать кэширование.
*   Обогатить схему данных полем segment.

### 3. Интеграция с CRM
*   Отправить в выбранную CRM две задачи на основе обогащенных данных:
    1.  Топ-5 самых дорогих товаров в сегменте Премиум.
    2.  Топ-5 самых дешевых товаров в сегменте Эконом.
*   Задачи создаются через официальный REST API CRM (с авторизацией, обработкой ошибок и логированием ответов).
*   В описании задачи в CRM должен быть список товаров с их URL и ценой.

### 4. Mock-режимы
*   Предусмотреть MOCK_CRM=true и MOCK_LLM=true для локального тестирования и прогона CI/CD без реальных лимитов внешних сервисов.

##  Технические требования

*   Валидация: Строгая типизация и валидация схем данных через Pydantic.
*   Тестирование: pytest, покрытие кода ≥70%. Обязательны моки внешних HTTP-запросов (парсинг, LLM, CRM).
*   AI-лог: Файл AI_USAGE.md с полным логом взаимодействия с ИИ-агентом (какие промпты использовались, как решалась задача батчинга, что сгенерировано ИИ, а что исправлено вручную, архитектурные решения).
*   Контейнеризация: docker-compose.yml для поднятия приложения (и опционально mock-сервисов, если нужно).
*   CI/CD: GitHub Actions (или GitLab CI). Минимальный пайплайн: линтер → тесты → сборка Docker-образа.
*   Документация: README.md (как установить, переменные окружения .env.example, как запустить, описание архитектуры, известные ограничения).
